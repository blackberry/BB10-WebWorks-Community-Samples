You don't know it yet, but you just learned how to initiate a Welcome Screen using localStorage!
If you've ever looked at the standard bbUI sample created by Tim Niel, N. Adam Stanley, and other RIM bbUI employees and volunteers, you might have noticed the following line:
Well, in order to have the app display a welcome screen, we're not using that onload command. Instead, we're using this: That allows us to call a JavaScript function when the app launched, and letting the JavaScript tell our app what to do. Specifically, the welcome() function does the following:
function welcome() {
if (localStorage.getItem('welcome') === null) {
bb.pushScreen('welcome.html', 'welcome');}
else
bb.pushScreen('menu.html', 'menu');
}
If you already know a bit about JavaScript, you know that function is basically telling the app "if there's no value set for the localStorage key welcome (that is, it returns null), load welcome.html. If there is a value set for that key, load menu.html" (and if you don't already know some JavaScript, now you know what that function means).
Set localStorage for Welcome and launch the app.
